Metal3

Bare metal host provisioning for Kubernetes

Goals of Metal3


By creating a stack running on Kubernetes, Metal3 hopes to expand on the ironic technology already in place to offer a Kubernetes native API for managing bare metal. One of the main objectives of Metal3 is to provide a widely used method for managing the lifespan of bare metal hosts that is simple to use and integrates well with Kubernetes. Metal³ will also eliminate the requirement for specialised hardware management programs by offering APIs and tools for managing bare metal hosts.


Metal3 scales and manages large numbers of bare metal hosts as part of its goals. It is essential for users with large workloads who need to deploy projects using containers like Kubernetes and manage infrastructure across multiple data centres or regions.

Mission of Metal3


Metal³ addresses the difficulties in managing and updating bare metal hosts at scale, the requirement to manually provision and configure hardware and the absence of standardisation for hardware inventory and configuration. Metal3 makes installing and managing Kubernetes clusters on bare metal easier by offering tools and APIs for automating these processes.


The overall objective of Metal³ is to enable Kubernetes users to gain from the performance, dependability, and flexibility of bare metal infrastructure while also utilising Kubernetes' features, such as automated deployment and scaling, application portability, and a robust ecosystem of tools and services.

Components of Metal3

Bare Metal Operator?
The Kubernetes operator, known as the bare metal host opertor [BMH Operator], oversees the lifetime of bare metal hosts. It is responsible for finding bare metal hosts, installing an operating system, and controlling their configuration and updates.
Hardware Manager Operator?
This Kubernetes operator finds and makes bare metal host hardware inventory data available. Regardless of the hardware type or manufacturer, it offers Kubernetes a standardised way to access hardware data.
Cluster API Provider Metal³?
By utilising Metal3, this Kubernetes Cluster API provider enables Kubernetes to control the lifespan of bare metal hosts. For Kubernetes to establish and manage bare metal hosts as a component of a Kubernetes cluster, it offers an API and a set of tools.
Metal³ Bootstrap?
A Kubernetes cluster can be started on bare metal hosts with the aid of this tool. In addition, it offers a collection of templates and scripts for automatically setting up hosts, installing Kubernetes, and adding them to a group.
Metal³ Machine Config Operator?
This Kubernetes operator controls how the operating system is set up on bare metal hosts. It offers a mechanism to alter the hosts' networking, storage, and other settings. Moreover, it can repair and upgrade hosts as necessary.

About Us


A PaaS (platform as a service) that provides resources to allow for simple implementation, deployment and management of raw resources on metal servers. Metal3, pronounced metal cubed, is an open-source tool for creating and setting up the bare metal.
